# COMPLIANCE POLICY
## Vendor Security Gate — Mandatory Data Vendor Onboarding Checklist

**Author:** Rachel Moore, Compliance  
**Date:** 2021-12-01  
**Version:** 2.1 (updated 2023-06-14)  
**Status:** MANDATORY — All data-handling vendor engagements  
**Risk Level:** High  
**Department:** Compliance / Procurement  
**Outcome:** Zero vendor-related security incidents since adoption (2021–present)  

---

### Background

Following the DataVault Systems breach (October 2021, ICO fine £180,000), this policy was established to ensure no data-handling vendor begins work before security posture is independently verified.

This policy is non-negotiable. Timeline pressure is not grounds for exception.

---

### Vendor Security Gate — 5-Point Checklist

All five items must be confirmed BEFORE a vendor accesses any company or client data:

#### 1. Cyber Essentials Plus Certificate
- Must be current (not expired)
- Issued by a UKAS-accredited body
- Emma Patel to verify expiry date in procurement file

#### 2. ISO 27001 or SOC 2 Type II Certification
- Certificate must be in scope for the services being provided
- Expiry must be checked — **the DataVault incident occurred with an expired cert accepted at face value**
- Compliance to flag any certificate more than 18 months old

#### 3. Penetration Test Report
- Must be within 12 months of onboarding date
- Vendor-supplied is acceptable if conducted by an accredited CREST firm
- Critical or High findings must be remediated before approval

#### 4. Data Processing Agreement
- Must be reviewed and signed by Legal (Sarah Chen or delegated counsel) before work begins
- GDPR/UK GDPR compliance required
- Sub-processor list must be disclosed

#### 5. Compliance Sign-Off
- Rachel Moore (or designated deputy) must confirm all four above items before issuing Vendor Approved status
- No Vendor Approved status = no data access. No exceptions.

### Incident History

This policy was created in response to:
- **DataVault Systems Ltd breach, October 2021** — vendor onboarded without completing security checks. ICO fine: £180,000. Project delay: 6 months.

Since adoption of this policy (December 2021 through 2024), zero vendor-related security incidents have occurred.
